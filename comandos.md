sirven
- git init para inicializar GIT en el proyecto si no tiene el init no serviran ningun comando es dar inicio solo hacer una vez por proyecto ya que creara las carpetas necesarias para git funcione
- git add . Sirve para añadir todo al proyecto es decir copiar todos los archivos cambio etc.
- git commit -m "Nombre del cambio/Rama/ajustes" Esto Sirve para crear un nombre con los cambios que se subiran super necesario en caso de querer regresar a una version estable encontrarlo por los nombres del commit
es decir este proyecto funciona ahora, mañana ajustas y pum murio :c , con el nombre del commit eres capaz de regresar a donde estaba bien :D.
- git branch -M main Esto sirve solo en tu caso que eres lider y tienes que subir a github por que hay mas comandos pero esto en si es lo que manda a la rama principal es decir el main xd
por que digo rama?, por que en el proyecto puedes añadir muchas cosas y nunca afectar el main cuando haces ramas son muy utiles para cuando esta lista esa rama la fusionas con el main :3
- git remote add origin <https://github.com/AlissCastillo/Backend_Aliss.git> aqui va el url del github donde se hara todas las subidas cambios etc
